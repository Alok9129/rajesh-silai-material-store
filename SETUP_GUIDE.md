# 🚀 Complete Setup & Deployment Guide\n\nRajesh Silai Material Store - E-Commerce Platform\n\n---\n\n## 📋 Table of Contents\n1. [Clone & Run Locally](#-clone--run-locally)\n2. [Deploy to Heroku](#-deploy-to-heroku)\n3. [Deploy to Vercel](#-deploy-to-vercel)\n4. [Share with Team](#-share-with-team)\n5. [Make Changes & Push Updates](#-make-changes--push-updates)\n\n---\n\n## 1️⃣ Clone & Run Locally\n\n### Prerequisites\n- ✅ Node.js (v14+) - [Download](https://nodejs.org/)\n- ✅ MongoDB - [Download](https://www.mongodb.com/try/download/community) OR use [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)\n- ✅ Git - [Download](https://git-scm.com/)\n- ✅ VS Code (Optional) - [Download](https://code.visualstudio.com/)\n\n### Step 1: Clone Repository\n```bash\ngit clone https://github.com/Alok9129/rajesh-silai-material-store.git\ncd rajesh-silai-material-store\n```\n\n### Step 2: Setup Backend\n```bash\ncd backend\nnpm install\ncp .env.example .env\n```\n\n### Step 3: Configure .env File\n\n**Option A: Using Local MongoDB**\n\nMake sure MongoDB is running, then edit `backend/.env`:\n```\nMONGODB_URI=mongodb://localhost:27017/rajesh-silai\nPORT=5000\nNODE_ENV=development\nJWT_SECRET=rajesh-silai-secret-key-2024\n```\n\n**Option B: Using MongoDB Atlas (Cloud)**\n\n1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)\n2. Create free account\n3. Create cluster\n4. Get connection string\n5. Update `backend/.env`:\n```\nMONGODB_URI=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/rajesh-silai?retryWrites=true&w=majority\nPORT=5000\nNODE_ENV=development\nJWT_SECRET=rajesh-silai-secret-key-2024\n```\n\n### Step 4: Start Backend Server\n```bash\ncd backend\nnpm start\n```\n\n✅ You should see:\n```\n✅ MongoDB connected successfully\n🚀 Server running on http://localhost:5000\n📍 API Health: http://localhost:5000/api/health\n```\n\n### Step 5: Start Frontend (New Terminal)\n```bash\ncd frontend\n\n# Option 1: Using Python (built-in)\npython -m http.server 3000\n\n# Option 2: Using Node\nnpx http-server -p 3000\n```\n\n### Step 6: Open in Browser\n- **Frontend**: http://localhost:3000\n- **API Health**: http://localhost:5000/api/health\n\n### 🎯 Test the Application\n\n1. **Home Page** - Browse products\n2. **Register** - Create new account\n   - Select \"Customer\" or \"Admin\"\n   - Fill in details\n   - Click Register\n3. **Login** - Sign in with credentials\n4. **Shop** - Add items to cart\n5. **Admin Panel** - Access at `/admin/dashboard.html` (if logged in as admin)\n\n---\n\n## 2️⃣ Deploy to Heroku\n\n### Prerequisites\n- ✅ Heroku Account - [Create Free Account](https://www.heroku.com/)\n- ✅ Heroku CLI - [Download](https://devcenter.heroku.com/articles/heroku-cli)\n- ✅ Git\n\n### Step 1: Login to Heroku\n```bash\nheroku login\n```\n\n### Step 2: Create Heroku App\n```bash\nheroku create your-app-name\n```\n\nReplace `your-app-name` with something like:\n- `rajesh-silai-store`\n- `rajesh-silai-backend`\n- `material-store-api`\n\n### Step 3: Set Environment Variables\n```bash\nheroku config:set MONGODB_URI=your_mongodb_atlas_uri\nheroku config:set JWT_SECRET=your-jwt-secret-key\nheroku config:set NODE_ENV=production\n```\n\n### Step 4: Deploy Backend to Heroku\n```bash\n# From project root\ngit push heroku main\n```\n\nWait for deployment to complete. You'll see:\n```\nremote: -----> Compressing source files... done.\nremote: -----> Building application...\nremote: -----> Discovering process types\nremote:        Procfile declares types -> web\nremote: \nremote: -----> Launching... done\n```\n\n### Step 5: Check Deployment\n```bash\nheroku open\n# or visit: https://your-app-name.herokuapp.com/api/health\n```\n\n### Step 6: View Logs\n```bash\nheroku logs --tail\n```\n\n### Deploy Frontend to GitHub Pages\n\n1. Frontend is automatically deployed at:\n   ```\n   https://Alok9129.github.io/rajesh-silai-material-store\n   ```\n\n2. Update API URL in `frontend/app.js`:\n   ```javascript\n   const API_URL = 'https://your-app-name.herokuapp.com/api';\n   ```\n\n3. Push changes:\n   ```bash\n   git add .\n   git commit -m \"Update API URL for production\"\n   git push origin main\n   ```\n\n---\n\n## 3️⃣ Deploy to Vercel\n\n### Prerequisites\n- ✅ Vercel Account - [Sign Up](https://vercel.com/signup)\n- ✅ GitHub Account connected to Vercel\n\n### Step 1: Create vercel.json for Backend\n\nCreate `vercel.json` in project root:\n```json\n{\n  \"version\": 2,\n  \"builds\": [\n    {\n      \"src\": \"backend/server.js\",\n      \"use\": \"@vercel/node\"\n    }\n  ],\n  \"routes\": [\n    {\n      \"src\": \"/(.*)\",\n      \"dest\": \"backend/server.js\"\n    }\n  ],\n  \"env\": {\n    \"MONGODB_URI\": \"@mongodb_uri\",\n    \"JWT_SECRET\": \"@jwt_secret\",\n    \"NODE_ENV\": \"production\"\n  }\n}\n```\n\n### Step 2: Push to GitHub\n```bash\ngit add .\ngit commit -m \"Add Vercel configuration\"\ngit push origin main\n```\n\n### Step 3: Deploy Backend to Vercel\n\n1. Go to [Vercel Dashboard](https://vercel.com/dashboard)\n2. Click \"New Project\"\n3. Select your GitHub repository\n4. Click \"Import\"\n5. Add Environment Variables:\n   - `MONGODB_URI` = your MongoDB Atlas URI\n   - `JWT_SECRET` = your secret key\n6. Click \"Deploy\"\n\n### Step 4: Deploy Frontend to Vercel\n\n1. Create `frontend/vercel.json`:\n```json\n{\n  \"rewrites\": [\n    { \"source\": \"/(.*)\", \"destination\": \"/\" }\n  ]\n}\n```\n\n2. Update API URL in `frontend/app.js`:\n```javascript\nconst API_URL = 'https://your-backend.vercel.app/api';\n```\n\n3. Push and create new Vercel project for frontend\n\n### Step 5: Get Deployment URLs\n\n- Backend: `https://your-backend.vercel.app`\n- Frontend: `https://your-frontend.vercel.app`\n\n---\n\n## 3️⃣ Share with Team\n\n### Option 1: Invite to GitHub Repository\n\n1. Example changes:\n\n**Add Review Feature:**\n1. Update `backend/models/Product.js`\n2. Add review routes in `backend/routes/products.js`\n3. Update frontend to display reviews\n4. Test everything locally\n\n### Step 3: Test Locally\n```bash\n# Terminal 1: Backend\ncd backend\nnpm start\n\n# Terminal 2: Frontend\ncd frontend\npython -m http.server 3000\n```\n\nTest at http://localhost:3000\n\n### Step 4: Commit Changes\n```bash\n# Check status\ngit status\n\n# Stage all changes\ngit add .\n\n# Commit with descriptive message\ngit commit -m \"Add feature: Product reviews\"\n# or\ngit commit -m \"Fix: Cart calculation bug\"\n# or\ngit commit -m \"Update: Improve search functionality\"\n```\n\n### Step 5: Push to GitHub\n```bash\ngit push origin feature/your-feature-name\n```\n\n### Step 6: Create Pull Request\n\n1. Go to GitHub repository\n2. Click \"Pull Requests\" tab\n3. Click \"New Pull Request\"\n4. Select:\n   - **Base Branch**: `main`\n   - **Compare Branch**: `feature/your-feature-name`\n5. Add description\n6. Click \"Create Pull Request\"\n\n### Step 7: Code Review & Merge\n\n1. Team members review code\n2. Add comments/suggestions if needed\n3. Once approved, click \"Merge Pull Request\"\n4. Delete feature branch\n\n### Step 8: Update Local Repository\n```bash\ngit checkout main\ngit pull origin main\n```\n\n---\n\n## 📝 Commit Message Guidelines\n\nUse clear, descriptive commit
