If it does:\ngit rm --cached .env\ngit commit -m \"Remove .env from tracking\"\ngit push origin main\n```\n\n### Step 4: Create Template File\n\n**File:** `backend/.env.example`\n\n```env\n# Database\nMONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/rajesh-silai?retryWrites=true&w=majority\n\n# Server\nPORT=5000\nNODE_ENV=development\n\n# Authentication\nJWT_SECRET=your-strong-random-jwt-secret-here\n\n# Payment Gateway\nRAZORPAY_KEY_ID=rzp_test_abc123def456xyz\nRAZORPAY_KEY_SECRET=rzp_test_secret_xyz789abc\n\n# Frontend URL\nFRONTEND_URL=http://localhost:3000\n```\n\n### Step 5: Share Template Only\n\n✅ DO share: `.env.example` (template only)\n❌ DO NOT share: `.env` (actual credentials)\n\n---\n\n## 🔄 Development vs Production\n\n### Development Environment\n\n**File:** `backend/.env` (local only)\n\n```env\n# Development\nMONGODB_URI=mongodb://localhost:27017/rajesh-silai\nPORT=5000\nNODE_ENV=development\nJWT_SECRET=dev-secret-key-change-in-production\n\n# Razorpay TEST 123\nOTP: 123456\n```\n\n### Production Environment\n\n**Platform:** Heroku/Vercel/Railway Config\n\n```env\n# Production\nMONGODB_URI=mongodb+srv://produser:prodpass@prod-cluster.mongodb.net/rajesh-silai?retryWrites=true&w=majority\nPORT=5000\nNODE_ENV=production\nJWT_SECRET=prod-secret-a7f9e2d1c4b8f3e9a2d7c5f1e8b9a3d6f2e4c7a\n\n# Razorpay LIVE MODE\nRAZORPAY_KEY_ID=rzp_live_abc123def456xyz\nRAZORPAY_KEY_SECRET=rzp_live_secret_xyz789abc\n```\n\n### Key Differences:\n\n| Aspect | Development | Production |\n|--------|-------------|------------|\n| JWT Secret | Simple (for testing) | Strong (32+ chars, random) |\n| Razorpay Mode | TEST (rzp_test_*) | LIVE (rzp_live_*) |\n| MongoDB | Local | MongoDB Atlas |\n| NODE_ENV | development | production |\n| Logging | Verbose | Minimal |\n| HTTPS | No | Yes (Required) |\n\n---\n\n## 💳 Razorpay Security\n\n### Test Mode Setup\n\n**Step 1: Use Test Keys**\n```env\nRAZORPAY_KEY_ID=rzp_test_abc123def456xyz\nRAZORPAY_KEY_SECRET=rzp_test_secret_xyz789abc\n```\n\n**Step 2: Razorpay Test Card Numbers**\n\n```\n✅ Successful Payment:\n- Card: 4111 1111 1111 1111\n- Expiry: Any future date\n- CVV: Any 3 digits\n- OTP: 123456\n\n✅ Failed Payment:\n- Card: 4222 2222 2222 2220\n- Expiry: Any future date\n- CVV: Any 3 digits\n```\n\n### Production Mode Setup\n\n**Step 1: Get Live Keys**\n- Login to Razorpay Dashboard\n- Settings → API Keys\n- Copy \"Key ID\" and \"Key Secret\"\n\n**Step 2: Add to Production Environment**\n```env\nRAZORPAY_KEY_ID=rzp_live_abc123def456xyz\nRAZORPAY_KEY_SECRET=rzp_live_secret_xyz789abc\n```\n\n**Step 3: Enable SSL/HTTPS**\n- Razorpay requires HTTPS for live mode\n- Your domain must have valid SSL certificate\n\n### Never Do This:\n\n```javascript\n// ❌ WRONG - Hardcoded keys\nconst RAZORPAY_KEY = 'rzp_live_abc123def456xyz';\n\n// ❌ WRONG - Committing to GitHub\n// backend/.env in git history\n\n// ❌ WRONG - Same keys everywhere\n// Production and development use same keys\n```\n\n### Always Do This:\n\n```javascript\n// ✅ CORRECT - Using environment variables\nconst RAZORPAY_KEY = process.env.RAZORPAY_KEY_ID;\nconst RAZORPAY_SECRET = process.env.RAZORPAY_KEY_SECRET;\n\n// ✅ CORRECT - .env in .gitignore\n// .env.example as template\n\n// ✅ CORRECT - Different keys per environment\n// Dev: rzp_test_*\n// Prod: rzp_live_*\n```\n\n---\n\n## 🗄️ MongoDB Credentials\n\n### MongoDB Atlas Connection String\n\n```\nFormat: mongodb+srv://username:password@cluster.mongodb.net/database?retryWrites=true&w=majority\n```\n\n### Step 1: Create MongoDB Atlas Account\n- Go to: https://www.mongodb.com/cloud/atlas\n- Sign up for free\n- Create organization and project\n\n### Step 2: Build Cluster\n- Select \"Free Tier\"\n- Choose region (closest to your users)\n- Create cluster (takes 2-5 minutes)\n\n### Step 3: Create Database User\n- Go to Database Access\n- Create new user\n- Set username: `rajeshsilaiuser` (example)\n- Set password: Auto-generate strong password\n- Copy password (you won't see it again!)\n\n### Step 4: Whitelist IP Address\n- Go to Network Access\n- Add IP Address\n- Add `0.0.0.0/0` (allow all - for development)\n- For production: Add specific IP only\n\n### Step 5: Get Connection String\n- Click \"Connect\"\n- Select \"Drivers\" → Node.js\n- Copy connection string\n- Replace `<username>` and `<password>`\n- Replace `<database>` with `rajesh-silai`\n\n### Example:\n```env\n# Before:\nMONGODB_URI=mongodb+srv://<username>:<password>@cluster0.abc123.mongodb.net/<database>?retryWrites=true&w=majority\n\n# After:\nMONGODB_URI=mongodb+srv://rajeshsilaiuser:P@ssw0rd123@cluster0.abc123.mongodb.net/rajesh-silai?retryWrites=true&w=majority\n```\n\n### URL Encode Special Characters\n\nIf password contains special characters:\n\n```\n! = %21\n@ = %40\n# = %23\n$ = %24\n% = %25\n& = %26\n: = %3A\n/ = %2F\n```\n\nExample:\n```\nPassword: P@ss#123!&\nEncoded: P%40ss%23123%21%26\n```\n\n---\n\n## 🔄 compromise suspected\n\n### How to Rotate JWT Secret\n\n```bash\n# 1. Generate new secret\nnode -e \"console.log(require('crypto').randomBytes(32).toString('hex'))\"\n\n# 2. Update in production\nheroku config:set JWT_SECRET=\"new-secret-here\"\n\n# 3. Deploy changes\ngit push heroku main\n\n# 4. Update local .env (optional - for testing)\n# backend/.env → JWT_SECRET=new-secret-here\n\n# 5. Alert users about re-login if needed\n```\n\n---\n\n## ✅ Security Checklist\n\n### Before Development\n- [ ] Created strong JWT secret (32+ chars)\n- [ ] Created `.env` file locally\n- [ ] Added `.env` to `.gitignore`\n- [ ] Created `.env.example` template\n- [ ] Verified `.env` not in git history\n- [ ] Set up MongoDB Atlas\n- [ ] Created MongoDB user with strong password\n- [ ] Whitelisted IP address\n- [ ] Tested local connection\n\n### Before Deploying to Staging\n- [ ] All credentials different from production\n- [ ] Using Razorpay TEST keys (rzp_test_*)\n- [ ] JWT secret is production strength\n- [ ] .env NOT included in deployment\n- [ ] Environment variables set on platform\n- [ ] SSL certificate enabled\n- [ ] CORS properly configured\n- [ ] Logging enabled for debugging\n\n### Before Going to Production\n- [ ] Changed all TEST keys to LIVE keys\n- [ ] Updated MongoDB to production cluster\n- [ ] Generated new JWT secret for production\n- [ ] Configured all environment variables\n- [ ] Enabled HTTPS/SSL\n- [ ] Set up uptime monitoring\n- [ ] Configured backup schedule\n- [ ] Set up error logging\n- [ ] Team trained on security practices\n- [ ] Emergency response plan ready\n\n### Ongoing Security\n- [ ] Regular security audits (monthly)\n- [ ] Credential rotation (quarterly)\n- [ ] Database backups verified (daily)\n- [ ] Access logs reviewed (weekly)\n- [ ] No hardcoded credentials in code\n- [ ] Dependencies updated regularly\n- [ ] Security patches applied immediately\n\n---\n\n## 🚨 Emergency Response\n\n### If Credentials Are Compromised\n\n**Immediate Actions (Within 1 hour):**\n\n1. **Stop the server** if severely compromised\n   ```bash\n   heroku ps:stop web\n   ```\n\n2. **Rotate all credentials immediately**\n   - Generate new JWT secret\n   - Change MongoDB password\n   - Regenerate Razorpay keys\n   - Update all environment variables\n\n3. **Review access logs**\n   ```bash\n   # Heroku logs\n   heroku logs --tail\n   \n   # MongoDB Atlas logs\n   # Dashboard → Logs\n   ```\n\n4. **Notify stakeholders**\n   - Inform team members\n   - Alert administrators\n   - Document incident\n\n**Short Term (Within 24 hours):**\n\n5. **Audit database**\n   - Check for unauthorized changes\n   - Review recent transactions\n   - Verify data integrity\n\n6. **Force user re-authentication**\n   - Invalidate all existing JWT tokens\n   - Force users to login again\n\n7. **Review code history**\n   ```bash\n   git log --all --source --full-history -- backend/.env\n   git log -p -- backend/.env\n   ```\n\n8. **Remove compromised credentials from git**\n   ```bash\n   # Using BFG Repo-Cleaner\n   bfg --delete-files .env\n   git reflog expire --expire=now --all\n   git gc --prune=now --aggressive\n   git push origin --force\n   ```\n\n**Long Term (Within 1 week):**\n\n9. **Conduct security audit**\n10. **Update security policies**\n11. **Train team on security**\n12. **Implement additional monitoring**\n\n### If .env Accidentally Committed\n\n```bash\n# 1. Remove from git history\ngit rm --cached backend/.env\n\n# 2. Add to .gitignore\necho \"backend/.env\" >> .gitignore\n\n# 3. Commit removal\ngit add .gitignore\ngit commit -m \"Remove .env from git history\"\n\n# 4. Force push (only if not yet shared)\ngit push origin main --force\n\n# 5. IMPORTANT: Rotate ALL credentials\n# The compromised credentials should be considered leaked\n```\n\n---\n\n## 📚 Reference Links\n\n- [MongoDB Atlas Security](https://docs.mongodb.com/atlas/security/)\n- [JWT Best Practices](https://tools.ietf.org/html/rfc8725)\n- [Razorpay Security](https://razorpay.com/security/)\n- [Node.js Security Best Practices](https://nodejs.org/en/docs/guides/security/)\n- [OWASP Security Cheat Sheet](https://cheatsheetseries.owasp.org/)\n\n---\n\n## 🎯 Summary\n\n### ✅ Best Practices Implemented:\n\n1. **Strong JWT Secrets**\n   - 32+ character random strings\n   - Cryptographically secure\n   - Different per environment\n\n2. **Local .env Only**\n   - Never committed to git\n   - Added to .gitignore\n   - Template available (.env.example)\n\n3. **Regular Rotation**\n   - Quarterly credential updates\n   - Immediate rotation if compromised\n   - Documented rotation schedule\n\n4. **Razorpay Security**\n   - TEST keys for development\n   - LIVE keys for production\n   - Never hardcoded\n\n5. **MongoDB Protection**\n   - Strong passwords\n   - IP whitelisting\n   - Atlas backup enabled\n\n6. **Emergency Preparedness**\n   - Incident response plan\n   - Credential rotation procedures\n   - Access logging enabled\n\n---\n\n**Status:** ✅ Production Ready\n\n**Last Updated:** June 27, 2024\n\n**Questions?** Check git logs or contact security team.\n"
